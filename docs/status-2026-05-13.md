- `master` remains ahead of and behind `origin/master`; validation reflects the local branch state.
- The GitHub Release workflow is validated by local cross-build and static inspection, but actual asset publication still requires a tag/manual Actions run with repository permissions.
- `dot macos audit --json` is currently a bootstrap bridge with diagnostics; real brew/MAS/apps/defaults facts still belong to the read-only audit goal.
- The files module currently backs up all managed files before a non-empty Chezmoi diff because Chezmoi diff output is intentionally not parsed or printed for safety.
- Non-file macOS modules still need audit/capture/apply implementations and fixtures before they can use `auto_apply`.
- macOS Tahoe-specific implementation details still need fresh verification against live Apple/Homebrew/mas behavior before coding modules.
